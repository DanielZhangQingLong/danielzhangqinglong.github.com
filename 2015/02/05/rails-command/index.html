Rails 常用命令(一)

Veröffentlicht am 2015-02-05   |   in Rails

总结 rails 的几个比较有用的命令.

Rails new

创建新的项目时:

该命令会自动生成测试, 但是很多人往往不喜欢使用默认的单元测试, 不生成测试的命令是

Rails server

启动 rails 中给开发使用的简单 web server WEBrick

常用 option

  • rails server -e 指定启动的环境(默认是 development 环境)

    上面的命令等效, 使用测试环境启动服务器

  • rails server -p 指定启动端口(默认是3000)

  • rails server -d 守护进程启动 rails (不阻塞命令行)

    如果我想要干掉这些进程该怎么办? 重启? 可以这样:

rails console

可以和 rails 应用进行交互, 不需通过浏览器就可以操作数据库, 可以方便做一些试验.

常用options

  • rails c -s 沙盒, 操作完之后会回退, 不会影响数据内容

  • rails c -e 指定启动环境, 与 rails server 类似

rails dbconsole

数据库命令行, 支持Mysql, PostgresSQL, SQLite

rails runner

rails 环境下 运行 ruby 代码
